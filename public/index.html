Graph overload rendering

This renders a random graph generated by a node.js websocket, which emits a data point every millisecond. This data is saved and a graph is rendered using d3. The amount of data is calibrated so that it becomes difficult for the browser to finish rendering in time. (Ofcourse this will vary with different machines.)
The graph is rendered in a separate loop, allowing for maximum control from the user interface. The graph is made inefficient on purpose – to experiment with event-based pseudo-threading.
